# PRD：一期导购培训与跨平台比价

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 电商 AI 实验室一期 |
| 场景范围 | 导购培训、跨平台比价 |
| 文档类型 | PRD |
| 当前阶段 | 一期试点 |
| 目标周期 | 90 天 |

## 2. 背景与目标

### 2.1 背景

当前团队希望建设一套可复用的 Agent 能力平台，先在两个高价值场景打穿：

- 线下门店导购培训
- 内部运营跨平台比价

这两个场景都有明显人工痛点：

- 培训依赖人工讲解和反复陪练，效率低，内容更新慢，评估不稳定
- 比价依赖运营人工搜索、人工理解复杂福利规则，耗时长，结论不统一

### 2.2 一期目标

- 建成导购培训“学、听、练、评、改”闭环
- 建成跨平台比价“采、比、析、审、用”闭环
- 沉淀知识、语音、采集、评分、审计等共用能力
- 用试点数据验证 ROI，为二期扩展提供依据

### 2.3 一期非目标

- 不做全场景 Agent 平台开放化发布
- 不做视频动作和表情分析
- 不做 AI 评分直接影响绩效
- 不做全量平台实时监控
- 不做自动化定价决策闭环

## 3. 用户角色

| 角色 | 核心诉求 | 使用频次 |
| --- | --- | --- |
| 导购 | 快速掌握商品卖点，完成练习并获得反馈 | 高 |
| 店长 | 查看导购训练结果，做辅导和复核 | 中 |
| 培训运营 | 配置商品课程、审核话术、维护知识内容 | 中 |
| 价格运营 | 快速获取跨平台价格与福利差异，辅助策略判断 | 高 |
| 类目运营 | 了解竞品策略与平台差异 | 中 |
| 实验室管理员 | 管理配置、查看试点效果、监控风险 | 中 |

## 4. 成功指标

### 4.1 导购培训

- 训练完成率 >= 70%
- 复训率 >= 40%
- 人均准备时长下降 >= 30%
- AI 与店长评分一致性 >= 0.75
- 导购卖点覆盖率平均提升 >= 20%

### 4.2 跨平台比价

- 单次分析耗时下降 >= 50%
- 标准字段抽取成功率 >= 85%
- 人工复核通过率 >= 80%
- 运营使用周活稳定增长
- 结论采纳率 >= 30%

## 5. 需求范围

### 5.1 场景一：导购培训

#### 5.1.1 用户故事

- 作为培训运营，我希望选择某个商品并生成训练内容，以便快速发布课程
- 作为导购，我希望看到和听到标准示范内容，以便熟悉商品介绍方式
- 作为导购，我希望录下自己的讲解并收到明确反馈，以便知道哪里讲错或不熟练
- 作为店长，我希望查看导购训练结果和证据，以便进行辅导

#### 5.1.2 功能模块

| 模块 | 功能点 | 优先级 |
| --- | --- | --- |
| 商品课程配置 | 选择商品、配置课程目标、设置必讲卖点 | P0 |
| 内容生成 | 生成学习卡、话术稿、模拟问答 | P0 |
| 内容审核 | 运营修订、版本发布、回滚历史版本 | P0 |
| 听练能力 | TTS 播放、语速控制、音色选择 | P0 |
| 练习提交 | 录音、重录、上传、练习记录 | P0 |
| 自动点评 | 转写、错误标注、卖点覆盖分析、停顿分析 | P0 |
| 评分报告 | 分维度评分、问题时间点、改进建议 | P0 |
| 店长复核 | 查看结果、写评语、确认复训 | P1 |
| 训练统计 | 完成率、均分、进步趋势 | P1 |

#### 5.1.3 核心流程

```text
培训运营配置课程
  -> AI 生成训练内容
  -> 运营审核发布
  -> 导购查看学习卡和语音示范
  -> 导购录音练习
  -> 系统自动转写和评分
  -> 导购查看反馈
  -> 店长复核和辅导
```

#### 5.1.4 关键页面

- 课程配置页
- 商品学习卡页
- 话术与示范语音页
- 导购练习录音页
- 练习结果与评分报告页
- 店长复核页
- 训练数据看板

#### 5.1.5 业务规则

- 训练内容发布前必须经培训运营审核
- 每个商品课程必须保留版本号和发布时间
- 评分必须输出分维度结果，不允许只给总分
- 每个错误必须关联证据句或时间点
- 一期评分仅用于辅导，不直接进入绩效系统

#### 5.1.6 验收标准

- 导购可以独立完成一次完整训练闭环
- 评分报告可定位至少 3 类问题：事实错误、漏讲卖点、流畅度问题
- 店长可查看每次练习的录音、转写、评分和建议
- 培训运营可回溯课程版本及修改记录

### 5.2 场景二：跨平台比价

#### 5.2.1 用户故事

- 作为价格运营，我希望录入我方商品和竞品链接，以便快速形成比价结果
- 作为类目运营，我希望系统自动标准化复杂福利字段，以便减少人工理解成本
- 作为运营，我希望看到证据截图和异常提示，以便做人工复核

#### 5.2.2 功能模块

| 模块 | 功能点 | 优先级 |
| --- | --- | --- |
| 任务创建 | 录入商品、平台链接、对比目标 | P0 |
| 页面采集 | 调用 OpenClaw 访问页面、读取字段、截图留证 | P0 |
| 字段标准化 | 到手价、券后价、满减、赠品、会员权益等统一口径 | P0 |
| 同款判断 | 识别同款、近似款、不可比 | P0 |
| 差异分析 | 价格和福利差异解释、经营建议 | P0 |
| 异常标注 | 缺失字段、不可比、采集失败、规则冲突 | P0 |
| 证据展示 | 页面截图、原始文本、采集时间 | P0 |
| 运营复核 | 修正字段、标注采用结果、提交备注 | P1 |
| 比价报表 | 使用量、采集成功率、采纳率 | P1 |

#### 5.2.3 核心流程

```text
运营创建比价任务
  -> 系统调用 OpenClaw 采集页面
  -> 字段标准化与清洗
  -> 同款或近似款判断
  -> 价格和福利差异分析
  -> 风险与质量校验
  -> 输出对比报告
  -> 运营复核和采用
```

#### 5.2.4 关键页面

- 比价任务创建页
- 采集状态页
- 结构化对比报告页
- 证据与截图页
- 运营复核页
- 试点效果看板

#### 5.2.5 业务规则

- 平台采集数据必须保留采集时间和页面证据
- 无法判断的字段必须标记为不确定，不允许模型猜测
- 同款判断需明确标注为同款、近似款、不可比三类
- 系统生成的经营建议必须支持人工复核
- 一期不承诺全平台全量实时采集

#### 5.2.6 验收标准

- 运营可在一次任务中完成至少 2 个平台的对比
- 系统可输出标准化字段和差异解释
- 结果页可查看截图、原始字段、异常提示
- 对采集失败或不可比任务有清晰失败原因

## 6. 权限与角色控制

| 角色 | 权限 |
| --- | --- |
| 导购 | 查看课程、收听示范、提交练习、查看个人结果 |

| 培训运营 | 配置课程、编辑和发布内容、查看全局训练数据 |
| 价格运营 | 创建比价任务、查看报告、提交复核意见 |
| 管理员 | 配置平台、查看审计、管理模型与规则 |

## 7. 数据埋点

### 7.1 导购培训埋点

- 课程打开
- 音频播放开始/完成
- 录音开始/提交
- 评分报告打开
- 复训发起
- 店长复核完成

### 7.2 跨平台比价埋点

- 比价任务创建
- 采集成功/失败
- 字段标准化完成
- 报告查看
- 运营复核提交
- 建议采用/不采用

## 8. 风险与约束

- 训练内容错误会直接影响导购表达，必须有审核机制
- 录音质量差会影响评分准确性，需要提示录音环境要求
- OpenClaw 采集受页面结构和反爬策略影响，必须设计降级策略
- 平台福利口径复杂，标准化规则需要人工和业务共同校准
