# 一期汇报稿：导购培训 + 跨平台比价

## Page 1：一期目标和结论

标题建议：

`电商 AI 实验室一期方案：导购培训与跨平台比价`

核心表述：

- 一期不追求大而全，只做两个高价值场景
- 目标不是 Demo，而是形成可复用的 AI 能力底座
- 90 天内完成试点闭环和平台化雏形

一句话结论：

`先打穿导购培训和跨平台比价两个闭环，沉淀可编排 Agent 能力，再决定二期扩展。`

## Page 2：为什么只做这两个场景

建议标题：

`为什么一期聚焦两个场景`

可讲内容：

- 两个场景都有明确业务痛点和清晰用户
- 两个场景都能快速证明 ROI
- 两个场景共享大量底层能力
- 能以较小组织成本做试点验证

建议金句：

`先打穿，不铺开；先闭环，不摊大饼。`

## Page 3：一期整体架构

建议标题：

`一期不是多 Agent 聊天，而是能力平台`

页面结构建议：

- 左侧：导购培训流程
- 右侧：跨平台比价流程
- 底部：共用能力底座

要点：

- 顶层是业务场景和页面
- 中间是 Orchestrator Agent 和领域 Agent
- 底层是知识、语音、采集、规则、审计能力

建议金句：

`Agent 负责推理和编排，底层服务负责稳定与可信。`

## Page 4：场景一，导购培训闭环

建议标题：

`导购培训：从“会看资料”升级为“会讲、会练、会改”`

可讲流程：

1. 运营配置商品和课程目标
2. AI 生成训练卡和话术
3. 导购看和听示范内容
4. 导购录音练习
5. AI 给出逐条点评和分维度评分
6. 店长复核和二次辅导

建议突出：

- 一期先做音频，不做视频
- 一期先做辅导，不直接进绩效
- 每个评分都要给证据

## Page 5：场景二，跨平台比价闭环

建议标题：

`跨平台比价：从“人工搜、人工算”升级为“采、比、析、审”`

可讲流程：

1. 运营输入商品或链接
2. 系统采集竞品页面
3. AI 标准化价格和福利字段
4. AI 输出结构化对比表和经营建议
5. 运营人工复核和采用

建议突出：

- 不只比价格，还比券、赠品、会员权益、分期、以旧换新
- 结果必须附带页面证据和更新时间
- 一期目标是提升分析效率，不是自动定价

## Page 6：为什么用 OpenClaw

建议标题：

`没有接口时，OpenClaw 适合作为实验室阶段采集适配层`

可讲内容：

- 当前没有稳定数据接口，直接建设长期数据平台成本高
- OpenClaw 可驱动浏览器访问页面、抽取字段、保留截图证据
- 适合一期做“半自动、可复核”的比价试点

必须同步讲清楚：

- 它不是长期数据底座
- 页面结构变化、登录状态、反爬会影响稳定性
- 二期如果验证成立，要替换为更稳定的数据通道

建议金句：

`OpenClaw 解决的是“先跑通”，不是“永久依赖”。`

## Page 7：90 天路线图

建议标题：

`90 天路线图：先 MVP，再试点，再平台化`

建议拆成三段：

- 0 到 30 天：定义标准和打通底层能力
- 31 到 60 天：上线两个 MVP 并开始试点
- 61 到 90 天：做影子校准、平台化整理、复盘决策

建议强调：

- 培训评分先和人工并行
- 比价采集字段先抽检
- 每周都要有指标复盘

## Page 8：资源诉求和决策点

建议标题：

`需要的资源与管理决策`

建议内容：

- 产品：1 人，负责两个场景方案与推进
- 技术：后端、前端、移动端、算法各至少 1 人
- 数据和运营：各 1 人对口支持
- 试点门店和试点类目要提前锁定

建议让领导拍板的三件事：

- 是否同意一期只做两个场景
- 是否同意导购评分先不进入绩效
- 是否同意比价场景先采用 OpenClaw 作为实验室采集适配层
